## 1. МЕТА ТА ЗАВДАННЯ РОБОТИ

Метою лабораторної роботи є практичне застосування знань з розгортання програмних систем шляхом розміщення розробленої системи автоматизації процесів доставки на хмарній інфраструктурі Google Cloud Platform, налаштування всіх компонентів системи та перевірка їх коректної роботи.

## 2. ОПИС ПРОГРАМНОЇ СИСТЕМИ

### 2.1 Загальна характеристика

**RobDeliveryAPI** — програмна система для комплексної автоматизації процесів доставки "останньої милі" з використанням автономних роботизованих платформ (роботів-кур'єрів, дронів). Система забезпечує повний цикл управління замовленнями, оптимізацію маршрутів, моніторинг у реальному часі та безпечну передачу посилок.

### 2.2 Технологічний стек

**Backend:**
- ASP.NET Core 8.0 — серверний фреймворк
- Entity Framework Core — ORM для роботи з базою даних
- SQLite — система управління базами даних
- JWT (JSON Web Tokens) — автентифікація та авторизація
- SignalR — real-time комунікація для оновлення карти

**Frontend (тестовий клієнт):**
- React + Vite — фреймворк для побудови UI
- TypeScript — типізована надбудова над JavaScript
- Leaflet — бібліотека для інтерактивних карт

**IoT/Embedded:**
- ESP32/Arduino — мікроконтролери для дронів
- HTTP Client — комунікація з API
- GPS модулі — визначення координат
- Сенсори батареї — моніторинг заряду

**Інфраструктура:**
- Docker — контейнеризація додатку
- Google Cloud Platform — хмарна інфраструктура
- Google Compute Engine — віртуальні машини
- Linux (Debian) — операційна система сервера

### 2.3 Основні компоненти системи

1. **REST API Server** — обробка запитів, бізнес-логіка, управління даними
2. **Database** — збереження інформації про користувачів, замовлення, роботів, вузли
3. **Web Client** — інтерфейс для користувачів та адміністраторів
4. **IoT Devices** — автономні дрони та роботи-кур'єри
5. **Real-time Hub** — SignalR hub для live-оновлень карти

## 4. НАЛАШТУВАННЯ КОМПОНЕНТІВ

### 4.1 Налаштування серверної частини

**JWT Authentication**

Система використовує JWT для автентифікації користувачів та IoT пристроїв. Налаштування знаходяться в `Program.cs`:

```csharp
// Конфігурація JWT Authentication
var jwtKey = config.Jwt.Key;
var jwtIssuer = config.Jwt.Issuer;
var jwtAudience = config.Jwt.Audience;

builder.Services.AddAuthentication(options =>
{
    options.DefaultAuthenticateScheme = JwtBearerDefaults.AuthenticationScheme;
    options.DefaultChallengeScheme = JwtBearerDefaults.AuthenticationScheme;
})
.AddJwtBearer(options =>
{
    options.TokenValidationParameters = new TokenValidationParameters
    {
        ValidateIssuer = true,
        ValidateAudience = true,
        ValidateLifetime = true,
        ValidateIssuerSigningKey = true,
        ValidIssuer = jwtIssuer,
        ValidAudience = jwtAudience,
        IssuerSigningKey = new SymmetricSecurityKey(
            Encoding.UTF8.GetBytes(jwtKey)),
        ClockSkew = TimeSpan.Zero
    };
});
```

**CORS Configuration**

Для забезпечення взаємодії з веб-клієнтом та IoT пристроями налаштовано CORS:

```csharp
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowAllOrigins",
        builder =>
        {
            builder.SetIsOriginAllowed(_ => true)
                   .AllowAnyMethod()
                   .AllowAnyHeader()
                   .AllowCredentials();
        });
});
```

**Database Configuration**

Підключення до SQLite бази даних:

```csharp
var connectionString = config.ConnectionStrings.DefaultConnection;

builder.Services.AddDbContext<MyDbContext>(options =>
    options.UseSqlite(connectionString));

// Автоматичне створення БД при запуску
using (var scope = app.Services.CreateScope())
{
    var dbContext = scope.ServiceProvider.GetRequiredService<MyDbContext>();
    dbContext.Database.EnsureCreated();
}
```

**SignalR для Real-time оновлень**

```csharp
builder.Services.AddSignalR();

// Реєстрація хабу
app.MapHub<MapHub>("/hubs/map");
```

#### 7.5.1 Симуляція дрона на ESP32/Arduino

```cpp
#include <WiFi.h>
#include <HTTPClient.h>
#include <ArduinoJson.h>

const char* ssid = "WiFi_SSID";
const char* password = "WiFi_Password";
const char* serverUrl = "http://35.192.164.131:5102";
String authToken = "";

void setup() {
    Serial.begin(115200);
    WiFi.begin(ssid, password);

    while (WiFi.status() != WL_CONNECTED) {
        delay(1000);
        Serial.println("Connecting to WiFi...");
    }
    Serial.println("WiFi connected!");

    authToken = authenticateRobot();
}

void loop() {
    updateRobotStatus();
    delay(10000);
}

String authenticateRobot() {
    HTTPClient http;
    http.begin(serverUrl + String("/api/User/robot-login"));
    http.addHeader("Content-Type", "application/json");

    String payload = "{\"serialNumber\":\"DRONE-001\",\"accessKey\":\"secret\"}";
    int httpCode = http.POST(payload);

    if (httpCode == 200) {
        String response = http.getString();
        DynamicJsonDocument doc(1024);
        deserializeJson(doc, response);
        return doc["token"].as<String>();
    }

    return "";
}

void updateRobotStatus() {
    if (authToken == "") return;

    HTTPClient http;
    http.begin(serverUrl + String("/api/Robot/status"));
    http.addHeader("Authorization", "Bearer " + authToken);
    http.addHeader("Content-Type", "application/json");

    float lat = 50.4501 + random(-100, 100) / 10000.0;
    float lon = 30.5234 + random(-100, 100) / 10000.0;
    float battery = 75.5 - random(0, 10) / 10.0;

    String payload = "{";
    payload += "\"batteryLevel\":" + String(battery) + ",";
    payload += "\"currentLatitude\":" + String(lat, 6) + ",";
    payload += "\"currentLongitude\":" + String(lon, 6);
    payload += "}";

    int httpCode = http.POST(payload);

    if (httpCode == 200) {
        Serial.println("Status updated: Battery=" + String(battery));
    } else {
        Serial.println("Update failed: " + String(httpCode));
    }
}
```

### 7.6 Тестування навантаження

#### 7.6.1 Стрес-тест API

**PowerShell скрипт для тестування:**

```powershell
# Stress test - 100 concurrent requests
$url = "http://35.192.164.131:5102/api/Order"
$token = "Bearer YOUR_TOKEN_HERE"

$jobs = @()
for ($i = 1; $i -le 100; $i++) {
    $job = Start-Job -ScriptBlock {
        param($url, $token, $num)

        $headers = @{
            "Authorization" = $token
            "Content-Type" = "application/json"
        }

        $body = @{
            name = "Test Order $num"
            description = "Load test"
            weight = 2.0
            productPrice = 100
            isProductPaid = $true
            recipientId = 2
            deliveryPayer = "Sender"
        } | ConvertTo-Json

        $response = Invoke-RestMethod -Uri $url -Method Post `
            -Headers $headers -Body $body

        return $response
    } -ArgumentList $url, $token, $i

    $jobs += $job
}

# Wait for all jobs to complete
$jobs | Wait-Job
$results = $jobs | Receive-Job

Write-Host "Completed: $($results.Count) / 100 requests"
```

- **Всього запитів:** 100
- **Успішних:** 98
- **Помилок:** 2 (timeout)
- **Середній час відповіді:** 245 ms

### 8.3 Виявлені проблеми та їх вирішення

**Проблема 1:** Timeout при ExecuteOrder з багатьма дронами

- **Симптоми:** При наявності >10 дронів запит займав >5 секунд
- **Причина:** Послідовний розрахунок маршрутів
- **Рішення:** Оптимізація алгоритму з кешуванням координат
- **Результат:** Час виконання зменшено до ~450 ms

**Проблема 2:** SignalR disconnect при бездіяльності

- **Симптоми:** Веб-клієнти втрачали з'єднання після 30 хвилин
- **Причина:** Firewall закривав неактивні з'єднання
- **Рішення:** Keep-alive ping кожні 15 секунд
- **Результат:** З'єднання стабільні >24 години

**Проблема 3:** Docker контейнер не зберігав БД

- **Симптоми:** Після рестарту дані губилися
- **Причина:** БД всередині контейнера без volume
- **Рішення:** Налаштування Docker volume
- **Результат:** БД персистентна між рестартами
